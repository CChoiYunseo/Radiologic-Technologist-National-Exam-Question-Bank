# 방사선사 국가고시 AI 문제은행 구축

---

## 1. 프로젝트 개요

- 방사선사 국가고시 전체 범위를 대상으로 하는 AI 문제은행 구축 계획
- 공식 출제범위, 문항 설계 기준, 국문법 기준, 전공 근거 자료를 함께 반영
- 텍스트 기반 이론·법규·응용 문항과 영상·장비 기반 판단형 문항을 하나의 문제은행 구조 안에서 관리
- 최종 목표는 RAG 검색, 문항 생성, 자동 검증, 전문가 검수, 문제은행 저장, 사용자 풀이 서비스까지 연결하는 것

---

## 2. 왜 필요한가

- 방사선사 국가고시는 과목 범위가 넓고 단원별 반복 학습용 문항 수요가 큼
- 기존 문제 중심 학습은 저작권, 최신성, 범위 편중 문제가 발생할 수 있음
- AI를 활용하면 단원별·난이도별·유형별 문항을 빠르게 생성할 수 있음
- 다만 의료·보건 계열 시험은 근거성, 정답 유일성, 법규 최신성, 표현 명확성이 특히 중요함

---

## 3. 핵심 설계 방향

- 기출문제 복제 방식이 아니라 전공 자료와 공식 기준을 근거로 문항 생성
- RAG 기반으로 관련 근거를 먼저 검색한 뒤 LLM이 문항 생성
- 생성 결과는 바로 저장하지 않고 Harness 검증과 사람 검수를 거침
- 모든 문항에 출제범위, 근거, 난이도, 문항 유형, 검증 결과를 남김
- 영상·장비 기반 문항도 같은 출제범위 체계에 연결하되 검증 기준은 더 엄격하게 운영

---

## 4. 문항 범위

- 이론·법규
  - 방사선이론
  - 의료관계법규
  - 개념형, 계산형, 법규형 문항
  - 법규 기준일과 조문 관리 필수

- 임상 응용
  - 방사선응용
  - 검사 절차, 검사 조건, 임상 응용 판단
  - CT, MRI, 초음파, 핵의학, 치료 관련 문항

- 실기 판단
  - 실기 판단형 객관식 문항
  - 영상, 장비, 품질관리, 검사 상황 기반 문항
  - 이미지 정합성, 사용 권한, 비식별 검토, 전문가 검수 필요

## 18. 영상·장비 기반 문항 구축 전략

- 텍스트 RAG만으로 충분하지 않으므로 이미지 메타데이터와 검수 기준이 함께 필요
- 완전 생성형 의료영상보다 실제 비식별 자료, 공개 교육용 이미지, 시뮬레이션 이미지 활용이 더 안전함
- AI 생성 이미지는 해부학, 병변, 영상물리 오류 가능성이 높아 검증 부담이 큼
- 영상·장비 문항은 이미지 자체의 정합성 검증이 문항 검증만큼 중요함

```text
비식별·사용권한 확인
↓
영상 유형 및 출제범위 태깅
↓
이미지 메타데이터와 설명 텍스트 생성
↓
전문가 검수
↓
문항 템플릿 적용
↓
정답 유일성·이미지 정합성 검증
↓
문제은행 저장
```

## 20. 핵심 한계

- 저작권 리스크
  - 기출문제, 복원문제, 시중 모의고사 원문은 복제 위험이 큼
  - 전공 자료도 사용 권한과 출처를 관리해야 함
- LLM 할루시네이션
  - 근거에 없는 설명, 수치, 법령 조항을 만들 수 있음
  - RAG 근거가 부정확하면 생성 결과도 흔들림
- 정답 유일성
  - 5지선다 문항에서 정답이 2개 이상으로 해석될 수 있음
  - 오답이 너무 명백하면 문항 품질이 낮아짐
- 법규 최신성
  - 의료관계법규는 개정 가능성이 높음
  - 법령명, 조문, 기준일 관리가 필요함
- OCR·PDF·RAG 품질
  - 스캔본 PDF, 깨진 표, 부정확한 chunk 태깅은 잘못된 문항 생성을 유도할 수 있음
- 영상·장비 자료 검증
  - 의료영상 개인정보와 저작권 리스크
  - 해부학적 구조, 병변, 장비 형태 오류 가능성
  - 영상 판독 수준이 국가고시 범위를 넘어설 위험

---

## 21. 대응 방향

- 공식 출제범위와 사용 권한이 확인된 전공 자료 중심으로 생성
- 기출/모의고사 원문은 RAG 근거 자료로 등록 금지
- 문항마다 `evidence_refs`를 필수화
- 법규형 문항은 `law_name`, `article_ref`, `effective_date` 필수
- Harness는 1차 필터로 사용하고 전문가 검수 단계 필수화
- 영상형 문항은 이미지 정합성, 비식별, 정답 유일성 rubric을 추가로 적용
- 사용자 신고, 오답률, 검수 이력을 기반으로 재검토 루프 구축

---

## 22. 현재 남은 문제점

- 전공·법규·영상·장비 자료가 아직 충분히 업로드되지 않아 실제 RAG/이미지 기반 검색은 미구축
- `source_document`와 `source_chunk` 테이블은 모델만 있고 추출 파이프라인은 미구현
- 벡터 DB는 아직 선택 전
- 관리자 UI는 아직 없음
- 문항 생성 LLM API 연결 전
- 전문가 검수 워크플로우 미구현
- 사용자 풀이 화면과 오답노트 미구현
- 영상 자료 확보, 비식별 검토, 이미지 검증 체계 미구현

## 25. 결론
