# Quality Requirements

본 프로젝트는 방사선사 국가고시 1·2교시 문제은행 구축을 목표로 하며, 문제 생성보다 검증과 근거 연결을 우선한다.

## 할루시네이션 방지

- 문제는 반드시 RAG 검색 결과를 근거로 생성한다.
- RAG 근거에 없는 내용은 추가 생성하지 않는다.
- 해설은 근거 문서에 포함된 내용만 사용한다.
- 법규 문항은 법령명, 조문번호, 기준일을 저장한다.
- 수치, 단위, 공식은 원문과 비교 검증한다.
- 학습목표와 출제범위를 벗어난 문제는 생성하지 않는다.

## 저작권 보호

- 원문 문장, 원문 문제, 보기 구성, 해설을 그대로 복제하지 않는다.
- 출처와 페이지는 메타데이터로만 저장한다.
- RAG는 정답 근거 제공 목적으로만 사용한다.
- 문제, 보기, 해설은 새롭게 생성한다.
- 생성 결과가 원문과 높은 유사도를 보이면 폐기한다.

## 검증 에이전트

- 출제범위 검증: 문제와 학습목표가 공식 범위에 포함되는지 확인한다.
- 정답 유일성 검증: 정답이 정확히 하나인지 확인한다.
- RAG 근거 검증: 문제와 해설이 근거 문단에서 지원되는지 확인한다.
- 저작권 위험 검증: 원문 복제, 보기 재현, 해설 복제를 확인한다.
- 국문법 검증: 문항줄기, 답가지, 부정문, 중의성, 표기 오류를 확인한다.

## Harness 필수 통과 조건

- 보기 개수 = 5
- 정답 개수 = 1
- 해설 존재
- 출처 존재
- 출제범위 존재
- 학습목표 존재
- 근거 문서 존재
- 국문법 오류 없음
- 중복 문항 아님

## 상태 관리

- `generated`: 생성 완료, 검증 통과 전
- `reviewed`: 검증 에이전트와 Harness 통과
- `approved`: 최종 승인
- `rejected`: 폐기

최종 문제 DB에는 `reviewed` 또는 `approved` 상태만 저장한다.

## RAG 데이터 품질

- MVP에서는 텍스트 레이어 추출, OCR 기반 텍스트 추출, 문서 정제, chunk 생성, RAG 구축을 우선한다.
- 표 내부 텍스트, 그림 설명, 장비 구조도 설명은 별도 Phase로 분리한다.
- 표와 그림 때문에 전체 RAG 구축이 지연되지 않도록 한다.
- OCR 결과를 검증한다.
- 문서 구조를 검증한다.
- 표는 OCR 결과를 사용하지 않고 구조화된 Table Extraction 결과를 우선 사용한다.
- 그림, 도식, 장비 구조도는 Vision LLM 또는 사람 검수 설명문을 별도 메타데이터로 저장한다.
- OCR 결과만으로 그림과 표의 의미를 확정하지 않는다.
- 학습목표와 출제범위를 연결한다.
- 출처 및 페이지 정보를 유지한다.
- 원문을 대체하는 2차 PDF를 생성하지 않는다.
- 원문은 근거 자료로 유지하고 AI는 구조화와 검색 보조 용도로만 사용한다.
- Vision LLM으로 생성한 그림 설명은 원문 대체물이 아니라 검색 보조 메타데이터이며, 검수 전 정답 근거로 단독 사용하지 않는다.
